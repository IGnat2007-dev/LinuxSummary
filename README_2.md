Работа с текстом. Команды sed,awk,cut,sort,uniq,wc,xargs

Команда sed предназначена для работы с текстом.Она может:
* заменять текст sd "s/disable/enable/g" - заменить все слова со второго на первое для того чтобы сохранить эти изменения нужно добавить флаг -i. 
* удалять текст output | sed '/error/d' -удалит все строки в которых есть слово error
* Оформление вывода ps aux | grep gnome-clocks | awk "{print: $2}" | sed 's/^/Process ID: /' - команда sed здесь выводит текст с ID 
 
Команда awk позволяет работать с колонками и таблицами. 
Для этой команды даже строка явл. табл. и каждое слово явл. колонкой и имеет свой номер $1,$2.... Эта команда пригождается когда надо вывести какую-то конкретную информацию с большого вывода как в прошлом примере мы вывели ID процесса. Или ls -l | awk '{print:$9, $5}' вывдет название файла или папки и ее размер 

Команда cut используется как резак, для взятия конкретной информации
Команда sort используется для того чтобы отсортировать список файлов,чисел,строк по алфавиту,по размеру,в обратном порядке и тд.
Команда uniq исользуется для того чтобы удалить одинаковые элементы чтобы они незанимали место 
Пример использования:last | cut -d " " -f 1 | sort | uniq -c | sort -n
Команда выведет какие пользователи чаще всего заходят в систему.

Команда wc исп. для того чтобы вывести количество строк (wc -l),количество слов (wc -w) и количество символов (wc -m)
Команда xargs исп. в кач. перевозчика для команд которые не могут принимать значения с пайпов . xargs сама приниимает значение с пайпа и перед его в кач аргумента другим командам.

Сервер

Сервер это физический компьютер который состоит из железа (hardware) и програмного обеспечения которое делает этот компьютер сервером.
ПО сервера состоит из:
* ОС для компьютера (на базе Linux)
* внутренних утилит ОС для подключений,защиты и тд
* ПО для отдельных целей и задач (программы) 

Основные типы серверов: 
- сервера баз данных (для структурирования и хранения данных)
- веб сервера (для хранения сайтов,страниц к ним)
- игровые сервера (для хранения состояния игры)
- файловые зранилища

Первичная настройка сервера: 
1. Создание пользователя
- когда сервер пустой то там есть только пользователь root и это не безопасно потому что под root можно стереть весь сервер или хакеры могу зайти на сервер 
2. настройка sudo
- добавленние только что созданного пользователся в список тех кто может выполнять команду sudo для работы от имени суперпользоателя.
3. создание ssh ключей и передача публичного серверу для коннекта
4. убрать вход по обычному паролю 
- чтобы хакеры не смогли взломать сервер убираем вход по обычному паролю и вместо этого вход будет выполняться только с помощью ssh ключей
5. поставить базовый фаерволл 
- фаерволл это защита портов сервера от посторонних лиц
